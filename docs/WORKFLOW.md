# QAmatomer 運用ワークフロー

## 目的

安定運用版と開発版の混在を避け、安全に機能追加を進めるための運用ルールです。

## 役割

- `QAmatomer_stable.html`: 利用者向けの安定版
- `QAmatomer_dev.html`: 新規機能の追加・検証用

## 推奨フロー

1. 新規機能は `QAmatomer_dev.html` へ実装する。
2. 動作確認（既存機能の回帰確認を含む）を行う。
3. 安定版へ取り込むときは、差分内容を `CHANGELOG.md` に記録する。
4. `QAmatomer_stable.html` へ反映してリリースする。

## 最低限のチェック項目

- 新規プロジェクト作成
- ノード追加/編集/削除
- 注釈追加/編集/削除
- 全文検索（`Ctrl+K` / `Cmd+K`）
- エクスポート/インポート


## 保存運用の推奨

- 初回利用時に「保存フォルダを接続」で保存先フォルダを設定
- 作業中は区切りごとに Ctrl/Cmd + S で手動保存
- 定期的に通常のエクスポートも取り、二重でバックアップ

- まとめ作成時にジャンルフォルダを設定して分類を統一する
- 保存先フォルダ内でもジャンル別サブフォルダに保存されることを前提に運用する
