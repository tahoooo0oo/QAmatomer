## QAmatomer v4（現在の安定版）

**追加**
- ジャンルフォルダ機能を追加（ホーム画面で分類/フィルタ、設定画面で編集）
- File System Access API を使った「保存フォルダ + プロジェクト別JSONの手動保存」を追加
- 手動保存時にローカル保存先にもジャンル別サブフォルダを作成
- ヘッダーに「保存フォルダを接続」「保存 (Ctrl/Cmd+S)」UIを追加
- ホーム画面に2ペインのフォルダビューを追加（左: フォルダ一覧 / 右: フォルダ内ページ一覧）
- ホーム画面で「フォルダ表示 / 全ファイル表示」を切り替え可能にした
- 既存ページカードからフォルダ移動（再分類）できるようにした
- フォルダビューから「このフォルダに新規ページ」を追加
- 手動並び替えモードで↑↓による任意順序調整を追加
- 並び順設定を保存し、再起動後も同じ順序で表示するようにした
- ホーム画面の並び順を選択可能にした（作成日/更新日/五十音/手動）
- 手動保存で出力した `annotator-qa-project-file` 形式JSONもホームのインポートで取り込めるよう修正

**追加**
- Undo / Redo（`Ctrl/Cmd+Z`, `Ctrl/Cmd+Shift+Z`）を追加

**ドキュメント**
- README に手動保存とUndo/Redoの使い方を追記
- `docs/WORKFLOW.md` に保存運用を追記

## QAmatomer v3


**主な変更**
- ノードの上下並び替えボタン（ツリー・フラットリスト両方）を追加
- 「段落を追加」時に新しい段落へ自動スクロール
- 注釈・子要素リストをデフォルト折りたたみに変更（ルートは展開）
- 段落タイトルへの Markdown・数式レンダリング対応
- 段落編集ジャンプボタン（閲覧ペインから直接その段落の編集位置へジャンプ）

**削除**
- ノード間リンク機能（複雑さに対してメリットが薄いため廃止）
- スナップショット HTML 生成機能（同上）

## QAmatomer v1（初期版）

LLMmatomer から Q&A ツリー専用ツールとして分離して作成。

**主な機能**
- Q&A ツリー構造（階層的なノード管理）
- テキスト選択による注釈・ハイライト
- 注釈から派生した子ノードの作成（多重階層）
- 再帰的なツリーナビゲーター（左ペイン）
- セクション形式の回答エディタ（タイトル＋本文の複数ブロック）
- ツリーの全展開・全折りたたみボタン
- 複数プロジェクト管理・JSON エクスポート/インポート

## LLMmatomer（前身ツール）

QAmatomer の元になった PDF テキスト注釈ツール。以下のバージョンが archive に保存されている。

| バージョン | 主な内容 |
|---|---|
| v1 | PDF テキスト選択注釈の基本実装 |
| v2 | KaTeX 数式対応の注釈、ブレッドクラムトレイル |
| v3 | 注釈ツリービュー、テキスト編集 UX 改善 |
| v4 | ダブルクリックでエディタの対応行へジャンプ |
| v5 | アウトラインナビゲーション（`#` / `##` 見出しベース）|
| lineAnnotation_experimental | 行ベースアンカーの実験的実装 |
