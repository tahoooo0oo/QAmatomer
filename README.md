# QAmatomer

Q&A のツリー構造で知識を整理するブラウザ完結型のメモツールです。

## 特徴

- **ツリー構造で整理** — 質問と回答を親子関係で階層化し、複雑な知識体系をわかりやすく管理できます
- **テキスト注釈** — 回答文中の任意のテキストを選択してハイライト・補足メモを追加できます。注釈から派生した子ノードも作れます
- **Markdown + 数式** — 回答は Markdown で記述でき、`$...$` / `$$...$$` で KaTeX 数式をレンダリングできます
- **全文検索** — `Ctrl+K` / `Cmd+K` でプロジェクト内の質問・回答・注釈を横断検索できます
- **ローカル完結** — データは `localStorage` に保存されます。サーバー不要でオフラインでも使えます（数式・アイコンは CDN を使用）
- **エクスポート / インポート** — JSON 形式でバックアップ・別環境への移行ができます

## 使い方

用途に応じて以下をブラウザで開くだけで動きます。インストール不要です。

```
安定運用: QAmatomer_stable.html
新機能追加・実験: QAmatomer_dev.html
```

## ファイルの役割

- `QAmatomer_stable.html` — 現在の安定運用版（既存利用向け）
- `QAmatomer_dev.html` — 新機能追加・検証用

運用の混乱を避けるため、機能実装後は `dev` で確認してから `stable` へ反映してください。



## Chrome拡張（dev起動ランチャー）

`chrome_extension/` に、`QAmatomer_dev.html` を起動するMV3拡張を追加しています。

### できること
- **サイドパネルで dev を開く（推奨）**: ChatGPTを見ながら横でQAmatomerを使える
- 新規タブで dev ページを開く（従来方式）
- 起動URLをオプション画面で保存（既定: `http://127.0.0.1:4173/QAmatomer_dev.html`）

### 使い方（開発）
1. このリポジトリ直下で dev ページを配信（例: `python3 -m http.server 4173 --bind 0.0.0.0`）
2. Chrome の `chrome://extensions` を開く
3. デベロッパーモードON → 「パッケージ化されていない拡張機能を読み込む」
4. `chrome_extension/` フォルダを選択
5. ChatGPTタブで拡張ポップアップを開き、**「サイドパネルで開く（推奨）」**を押す
- もし「`sidePanel.open() may only be called in response to a user gesture.`」が出る場合は、拡張ポップアップから直接「サイドパネルで開く」を押して起動してください（この手順で解消します）。

> 注: 現在の `QAmatomer_dev.html` は CDN スクリプトに依存しているため、拡張内でビルドを再実装するのではなく、ローカル配信URLをサイドパネル/タブへ読み込む方式にしています。

## 手動保存（フォルダ接続 + プロジェクト別JSON）

ブラウザのキャッシュ削除で `localStorage` が消える事故に備え、**保存フォルダへ手動保存**できるようにしました。

1. 画面右上の **「保存フォルダを接続」** を押して保存先フォルダを選択
2. 編集中のまとめで **「保存 (Ctrl/Cmd+S)」** を押す（または `Ctrl+S` / `Cmd+S`）
3. そのまとめ専用の JSON が、選んだフォルダ内の「ジャンルフォルダ/ファイル.json」に作成/更新されます

> まとめごとにファイルが分かれるため、復元や管理がしやすくなります。
> この機能は File System Access API 対応ブラウザで利用できます。未対応ブラウザでは従来どおり「エクスポート」を併用してください。

## ジャンルフォルダ機能

- 各まとめに「ジャンルフォルダ」を設定できます（例: 数学 / プログラミング / 語学）。
- ホーム画面は「フォルダ表示 / 全ファイル表示」を切り替えできます。
- フォルダ表示では、左にフォルダ一覧、右に選択フォルダ内ページ一覧を表示できます。
- 既存ページ一覧からもフォルダを変更でき、あとから分類し直せます。
- フォルダを開いた状態で「このフォルダに新規ページ」から追加できます。
- `Ctrl/Cmd+S` 保存時、ローカル保存先にも同名フォルダを作成して、その中へ保存します。


## 並び順（固定表示 + 手動並び替え）

ホーム画面の並び順はセレクタで切り替えでき、選択はブラウザ内に保存されます。

- 作成日（古い順）※デフォルト
- 作成日（新しい順）
- 最終更新順
- 名前（五十音順）
- 手動並び替え（↑↓で好きな順）

これにより、開くたびに順序が変わる問題を防ぎつつ、用途に応じた一覧管理ができます。

## Undo / Redo

- `Ctrl+Z` / `Cmd+Z` : 1つ前に戻す
- `Ctrl+Shift+Z` / `Cmd+Shift+Z` : やり直す

※ 入力欄を編集中はブラウザ標準のUndo/Redoを優先します。

## データ構造

| 概念 | 説明 |
|---|---|
| **プロジェクト** | まとめのひとまとまり。複数作成できます |
| **ノード** | 質問（タイトル）と複数のセクション（回答・解説）で構成される単位 |
| **セクション** | ノード内の回答ブロック。タイトルと Markdown 本文を持ちます |
| **注釈** | セクション内のテキストに付けるハイライト＋補足メモ |

## キーボードショートカット

| キー | 動作 |
|---|---|
| `Ctrl+K` / `Cmd+K` | 全文検索を開く / 閉じる |
| `Ctrl+S` / `Cmd+S` | 現在のまとめを保存フォルダへ手動保存 |
| `Ctrl+Z` / `Cmd+Z` | Undo |
| `Ctrl+Shift+Z` / `Cmd+Shift+Z` | Redo |
| `Escape` | 検索・選択メニューを閉じる |

## エクスポート形式

```json
  "version": 2,
  "exportedAt": 1234567890,
  "projects": [
    {
      "name": "プロジェクト名",
      "nodes": { ... },
      "annotations": [ ... ]
    }
  ]
}
```
